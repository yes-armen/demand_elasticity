# Дашборд локальной эластичности спроса

Интерактивный Streamlit-дашборд поверх калиброванной модели `base+market+geo` (`05`) и движка сценариев (`07`).
Методология: [`docs/dashboard_spec.md`](../docs/dashboard_spec.md).

## Запуск
```bash
.venv/bin/streamlit run dashboard/app.py
```
(в этой сессии: `! .venv/bin/streamlit run dashboard/app.py`)

Откроется на http://localhost:8501.

## Что показывает
- KPI: активные лоты, supply (м²), средневзвеш. цена, базовый/сценарный спрос (м²),
  ожидаемые продажи, выручка, сегментная эластичность.
- Кривые спроса (м²) и выручки по сетке Δ; **оптимум выручки — только через health-gate**
  (если спрос неэластичен, рекомендация цены не выдаётся).
- Форма спроса по `rel_price` (асимметрия: эластично ниже медианы / плоско выше) — факт vs модель.
- Позиция плановой цены в распределении предложения.
- Сценарная таблица (−15…+15%) и автоинтерпретация.

## Методологические допущения (важно)
- **Горизонт 30 дней**; `P(sold)` калибрована (isotonic).
- **Шок уровня цены** (при прочих равных, через единый `src.elasticity`): на Δ двигаются `price`,
  `price_no_discount`, `log_price`, `discount_abs`, `rel_price`; признаки изменения цены
  (`price_change_*`) и коллинеарные относительные (`rel_price_district`, `price_gap_to_p10`)
  заморожены, иначе эластичность завышается. Конкуренты не реагируют (частичное равновесие).
- Абсолютные величины (Σp, спрос, выручка, продажи) — **модельная оценка с динамической
  калибр-поправкой** (factor = Σфакт/Σp по labelable-срезам, считается под загруженную модель).
  На эластичность и проценты поправка не влияет.
- Эластичность **area-взвешенная** `E=ΔQ/(Q·Δ)`, на ±5% ≈ **−0.7…−0.8** (рынок в целом неэластичен).
- `P(sold)` = «продастся ИЛИ уйдёт из наблюдения» (конфаунд «обрыва» мал, но есть).
- Рекомендация цены выдаётся **только** при интерьерном максимуме выручки (revenue-gate): для
  эластичных сегментов/ЖК — да (есть оптимальная цена, ≈54 из 259 ЖК на опорном срезе), для
  неэластичного рынка в целом — нет. То есть дашборд диагностический + точечный прайс-советник,
  а не глобальный оптимизатор.

## Файлы
- `app.py` — UI (Streamlit).
- `engine.py` — шок цены (делегирует `src.elasticity`), скоринг, агрегаты, эластичность, revenue-gate.
- `reliability.py` — tiers надёжности по `q_base` + иерархия fallback.
